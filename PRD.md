# Miren Work Assistant Skill PRD

## 产品概述

**产品名称**: miren-work-assistant
**版本**: v1.0
**更新日期**: 2026-04-28

### 产品定位

一款集成于 Claude Code 的智能工作助手 Skill，帮助用户高效管理日常工作任务、邮件处理和工作汇报，提升工作效率和信息整合能力。

### 目标用户

- 需要管理多项工作任务的职场人士
- 需要定期撰写工作汇报的团队成员
- 使用 Microsoft 365 邮件系统的企业用户

---

## 功能模块

### 1. Todo 任务管理

#### 1.1 任务记录
- **功能描述**: 快速记录待办事项
- **触发方式**: `/work todo add <任务内容>`
- **支持特性**:
  - 任务优先级标记 (P0/P1/P2/P3)
  - 截止日期设置
  - 任务分类标签
  - 任务关联（关联邮件、会议等）

#### 1.2 任务查看
- **功能描述**: 查看和筛选任务列表
- **触发方式**: `/work todo list [筛选条件]`
- **筛选维度**:
  - 按状态：待办/进行中/已完成
  - 按优先级
  - 按截止日期
  - 按标签

#### 1.3 任务更新
- **功能描述**: 更新任务状态和内容
- **触发方式**: `/work todo update <任务ID>`
- **操作类型**:
  - 标记完成
  - 修改优先级
  - 更新截止日期
  - 添加备注

---

### 2. 工作规划

#### 2.1 日计划
- **功能描述**: 生成每日工作计划
- **触发方式**: `/work plan daily`
- **功能特性**:
  - 自动整合当日待办任务
  - 按优先级和预估时间排序
  - 建议时间块分配
  - 识别可能冲突的任务

#### 2.2 周计划
- **功能描述**: 生成周度工作规划
- **触发方式**: `/work plan weekly`
- **功能特性**:
  - 汇总本周重点任务
  - OKR/目标对齐检查
  - 工作量评估和建议
  - 里程碑提醒

#### 2.3 任务排版
- **功能描述**: 将任务格式化为美观的文档格式
- **触发方式**: `/work format [格式类型]`
- **支持格式**:
  - Markdown 表格
  - 甘特图（Mermaid）
  - 看板视图
  - 时间线视图

---

### 3. 邮件处理（基于 ms-365-mcp-server）

#### 3.1 邮件读取
- **功能描述**: 读取和搜索 Outlook 邮件
- **触发方式**: `/work email read [筛选条件]`
- **筛选条件**:
  - 时间范围
  - 发件人
  - 主题关键词
  - 未读/已读状态
  - 文件夹

#### 3.2 邮件摘要
- **功能描述**: 智能总结邮件内容
- **触发方式**: `/work email summary`
- **功能特性**:
  - 单封邮件摘要
  - 邮件线程总结
  - 提取行动项
  - 识别重要信息

#### 3.3 邮件分类
- **功能描述**: 自动分类邮件
- **触发方式**: `/work email categorize`
- **分类维度**:
  - 需要回复
  - 仅供参考
  - 待处理事项
  - 可忽略

每日晨报 (daily-morning-report)

• 执行时间: 每天 UTC 05:30 (本地时间根据 config.yaml 中 timezone 设置)
• 状态: ⏸️ 从未运行（等待首次执行）
• 任务内容:
  • 系统状态报告（AI 助理、MS365 邮箱、Telegram、Cron 任务）
  • 今日待办任务（按优先级分类）
  • 历史待办统计（完成率、按优先级/类型统计）
  • 过去24小时邮件摘要（新收/未读/重要/需回复）
  • 重要任务提醒（工单、紧急事项 、被@）

2️⃣ 邮件检查 (email-check)

• 执行时间: UTC 06:00-15:00 每小时（工作时间）
• 静默时间: UTC 16:00 - 次日 05:30（过夜邮件在晨报中汇总）
• 任务内容:
  • 检查最近1小时内的未读邮件
  • 筛选重要邮件（被@提到、工单更新、安全告警）
  • 忽略自动通知（垃圾邮件）
  • 发现重要邮件时用中文总结

定时任务时间表（UTC 时间，本地时间根据 config.yaml 中 timezone 设置）:

  | UTC 时间 | 任务 |
  |----------|------|
  | 05:30 | 🌅 每日晨报（系统状态+待办+过夜邮件） |
  | 06:00-15:00 | 📧 邮件检查（每小时） |
  | 14:00 | 📝 每日工作总结 |
  | 06:30 (周五) | 📊 周报生成 |
  | 16:00-05:00 | 🌙 静默时间（不检查邮件） |

### 4. 工作汇报

#### 4.1 晨报汇报
- **功能描述**: 生成每日晨会汇报
- **触发方式**: `/work report morning`
- **内容模板**:
```
## 今日计划
- [ ] 重点任务1
- [ ] 重点任务2

## 昨日进展
- [x] 完成项1
- [x] 完成项2

## 需要支持
- 问题/风险点
```

#### 4.2 日报总结
- **功能描述**: 生成每日工作日报
- **触发方式**: `/work report daily`
- **内容模板**:
```
## 今日完成
- 任务1：完成情况、产出物
- 任务2：完成情况、产出物

## 明日计划
- 计划任务1
- 计划任务2

## 问题与风险
- 遇到的问题及解决方案
```

#### 4.3 周报总结
- **功能描述**: 生成周度工作周报
- **触发方式**: `/work report weekly`
- **内容模板**:
```
## 本周重点工作
| 任务 | 计划 | 实际 | 状态 |
|------|------|------|------|
| xxx  | xx%  | xx%  | ✅/⏳ |

## 关键产出
- 产出物1
- 产出物2

## 下周计划
- 重点任务1
- 重点任务2

## 问题与建议
- 问题描述及建议方案
```

## 数据存储

### 本地存储结构
```
~/.miren-work/
├── config.yaml          # 配置文件
├── todos/
│   ├── active.json      # 活跃任务
│   └── archive.json     # 归档任务
├── reports/
│   ├── daily/           # 日报存档
│   └── weekly/          # 周报存档
└── cache/
    └── email_cache.json # 邮件缓存
```

### 配置文件示例
```yaml
# config.yaml
user:
  name: "用户姓名"
  team: "所属团队"

email:
  default_folder: "inbox"
  cache_days: 7

report:
  templates:
    morning: "default"
    daily: "default"
    weekly: "default"

todo:
  default_priority: "P2"
  categories:
    - "开发"
    - "会议"
    - "文档"
    - "其他"
```

## 交互流程

### 典型使用场景

#### 场景1：每日晨会准备
```
用户: /work report morning
助手: 正在生成晨报...
      [读取昨日完成任务]
      [读取今日待办任务]
      [生成晨报内容]
```

#### 场景2：邮件处理
```
用户: /work email read -n 5
助手: 正在读取最近5封邮件...
      [显示邮件列表]
用户: 总结第2封邮件
助手: [生成邮件摘要和行动项]
```

#### 场景3：周报撰写
```
用户: /work report weekly
助手: 正在生成周报...
      [汇总本周完成任务]
      [统计任务完成率]
      [整合关键产出]
      [生成周报文档]
```

## 附录

### A. 优先级定义

| 优先级 | 含义 | 处理时限 |
|--------|------|----------|
| P0 | 紧急重要 | 立即处理 |
| P1 | 重要 | 当日完成 |
| P2 | 一般 | 本周完成 |
| P3 | 低优先级 | 有空处理 |

### B. 状态定义

| 状态 | 含义 |
|------|------|
| pending | 待办 |
| in_progress | 进行中 |
| done | 已完成 |
| blocked | 阻塞 |
| cancelled | 已取消 |
