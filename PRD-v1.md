# Miren Work Assistant Skill PRD

**版本**: v1.0
**更新日期**: 2026-04-28
**时区**: 用户可配置（默认 Asia/Shanghai）

---

## 1. 产品概述

### 1.1 产品定位

一款集成于 Claude Code 的智能工作助手 Skill，提供：
- 自动化晨报与邮件监控
- 任务管理与工作规划
- 工作汇报生成

### 1.2 目标用户

- 需要管理日常工作任务的职场人士
- 使用 Microsoft 365 邮件系统的企业用户
- 需要定期撰写工作汇报的团队成员

### 1.3 技术依赖

| 依赖 | 用途 | 认证方式 |
|------|------|----------|
| ms-365-mcp-server | 邮件读取 | OAuth 2.0 (Mail.Read) |
| Telegram Bot | 消息推送 | Bot Token |
| Cron 调度 | 定时任务 | 系统级 |

### 2.2 邮件检查 (email-check)

| 属性 | 值 |
|------|------|
| 执行时间 | UTC 06:00-15:00 每小时 |
| 静默时间 | UTC 16:00 - 次日 05:30 |

> 本地时间根据 config.yaml 中 timezone 设置自动转换显示

**检查规则**:
- 检查最近1小时内的未读邮件
- 筛选重要邮件（被@提到、工单更新、安全告警）
- 忽略自动通知/垃圾邮件
- 发现重要邮件时用中文总结推送

**过夜邮件处理**: 静默时间内的邮件在次日晨报中汇总

### 2.5 周报生成 (weekly-report)

| 属性 | 值 |
|------|------|
| 执行时间 | 每周五 UTC 06:30 (本地时间根据 config.yaml 中 timezone 设置) |
| 任务ID | `weekly-report` |

**数据来源**:
- 本周所有每日工作总结（`~/.miren-work/data/reports/daily/`）
- 本周完成的待办任务
- 聚合邮件发送统计

### 2.6 定时任务时间表

定时任务基于 UTC 时间执行，本地时间根据 `~/.miren-work/config.yaml` 中配置的 `user.timezone` 显示。

| UTC 时间 | 任务 |
|----------|------|
| 05:30 | 🌅 每日晨报（系统状态+待办+过夜邮件） |
| 06:00-15:00 | 📧 邮件检查（每小时） |
| 14:00 | 📝 每日工作总结 |
| 06:30 (周五) | 📊 周报生成 |
| 16:00-05:00 | 🌙 静默时间（不检查邮件） |

> 示例：若配置 `timezone: "Asia/Shanghai"` (UTC+8)，则 UTC 05:30 = 本地 13:30

## 3. Todo 任务管理

### 3.1 任务记录

**触发方式**: `/work todo add <任务内容>`

**参数**:
| 参数 | 说明 | 示例 |
|------|------|------|
| `-p` | 优先级 (P0/P1/P2/P3) | `-p P1` |
| `-d` | 截止日期 | `-d 2026-04-30` |
| `-t` | 标签/分类 | `-t 开发` |
| `-r` | 关联 (邮件/会议ID) | `-r email:123` |

**示例**:
```bash
/work todo add "完成PRD文档" -p P1 -d 2026-04-30 -t 文档
```

### 3.2 任务查看

**触发方式**: `/work todo list [筛选条件]`

**筛选参数**:
| 参数 | 说明 | 示例 |
|------|------|------|
| `-s` | 状态 | `-s pending,in_progress` |
| `-p` | 优先级 | `-p P0,P1` |
| `-d` | 截止日期范围 | `-d today` / `-d this_week` |
| `-t` | 标签 | `-t 开发` |

### 3.3 任务更新

**触发方式**: `/work todo update <任务ID>`

**操作参数**:
| 参数 | 说明 | 示例 |
|------|------|------|
| `-s` | 更新状态 | `-s done` |
| `-p` | 修改优先级 | `-p P0` |
| `-d` | 更新截止日期 | `-d 2026-05-01` |
| `-n` | 添加备注 | `-n "已完成初稿"` |

## 4. 工作规划

### 4.1 日计划

**触发方式**: `/work plan daily`

**功能**:
- 自动整合当日待办任务
- 按优先级和预估时间排序
- 建议时间块分配
- 识别可能冲突的任务

### 4.2 周计划

**触发方式**: `/work plan weekly`

**功能**:
- 汇总本周重点任务
- OKR/目标对齐检查
- 工作量评估和建议
- 里程碑提醒

### 4.3 任务排版

**触发方式**: `/work format [格式类型]`

**支持格式**:
| 格式 | 说明 |
|------|------|
| `table` | Markdown 表格 |
| `gantt` | 甘特图（Mermaid） |
| `kanban` | 看板视图 |
| `timeline` | 时间线视图 |

## 5. 邮件处理

### 5.1 邮件读取

**触发方式**: `/work email read [筛选条件]`

**筛选参数**:
| 参数 | 说明 | 示例 |
|------|------|------|
| `-n` | 数量 | `-n 10` |
| `-f` | 文件夹 | `-f inbox` |
| `-t` | 时间范围 | `-t 24h` / `-t today` |
| `-from` | 发件人 | `-from boss@company.com` |
| `-s` | 主题关键词 | `-s "工单"` |
| `-u` | 仅未读 | `-u` |

### 5.2 邮件摘要

**触发方式**: `/work email summary [邮件ID]`

**功能**:
- 单封邮件摘要
- 邮件线程总结
- 提取行动项（Action Items）
- 识别重要信息和截止日期

### 5.3 邮件分类

**触发方式**: `/work email categorize`

**自动分类维度**:
| 分类 | 说明 |
|------|------|
| 🔴 需要回复 | 直接发给你、需要响应的邮件 |
| 🟡 待处理事项 | 包含任务或行动项 |
| 🔵 仅供参考 | CC邮件、通知类 |
| ⚪ 可忽略 | 自动通知、广告邮件 |

### 5.4 发送邮件分析

**触发方式**: `/work email sent [筛选条件]`

**功能描述**: 分析用户发送的邮件，用于工作总结

**筛选参数**:
| 参数 | 说明 | 示例 |
|------|------|------|
| `-t` | 时间范围 | `-t today` / `-t 24h` / `-t this_week` |
| `-n` | 数量限制 | `-n 50` |
| `--analyze` | 生成分析报告 | `--analyze` |

**分析维度**:
- **工作类型统计**: 按邮件内容分类统计
- **收件人分析**: 与谁沟通最多
- **时间分布**: 邮件发送时间分布
- **关键词提取**: 提取工作相关关键词

**输出示例**:

### 6.2 日报总结

**触发方式**: `/work report daily`

**数据来源**:
1. **发送邮件分析** - 读取当日 Sent Items，分析工作内容
2. **已完成任务** - 读取当日标记为 done 的待办任务
3. **邮件回复统计** - 统计回复邮件的数量和类型

**自动生成逻辑**:
```
1. 读取 ms-365-mcp-server 获取当日发送的邮件
2. 按工作类型分类（工单/代码审查/会议/沟通）
3. 提取邮件主题生成工作明细
4. 读取 todos 中今日完成的任务
5. AI 生成工作总结
```

**模板**:
```markdown

### 📧 邮件工作
| 类型 | 数量 | 占比 |
|------|------|------|
| 工单处理 | 5 | 33% |
| 代码审查 | 3 | 20% |
| 会议沟通 | 4 | 27% |
| 日常沟通 | 3 | 20% |

**工作明细**:
{{email_work_details}}

{{completed_todos}}

### 📝 AI 总结
{{ai_summary}}

### 📅 明日计划
{{tomorrow_plan}}

### ⚠️ 问题与风险
{{issues_and_risks}}
```

**手动触发参数**:
| 参数 | 说明 | 示例 |
|------|------|------|
| `-d` | 指定日期 | `/work report daily -d 2026-04-27` |
| `--no-email` | 不分析邮件 | `/work report daily --no-email` |
| `--export` | 导出格式 | `/work report daily --export md` |

## 7. 数据存储

### 7.1 目录结构

```
~/.miren-work/
├── config.yaml              # 配置文件
├── todos/
│   ├── active.json          # 活跃任务
│   └── archive.json         # 归档任务
├── reports/
│   ├── morning/             # 晨报存档
```

## 9. 优先级与状态定义

### 9.1 优先级

| 优先级 | 标识 | 含义 | 处理时限 |
|--------|------|------|----------|
| P0 | 🔴 | 紧急重要 | 立即处理 |
| P1 | 🟠 | 重要 | 当日完成 |
| P2 | 🟡 | 一般 | 本周完成 |
| P3 | 🟢 | 低优先级 | 有空处理 |

### 9.2 任务状态

| 状态 | 含义 |
|------|------|
| `pending` | 待办 |
| `in_progress` | 进行中 |
| `done` | 已完成 |
| `blocked` | 阻塞 |
| `cancelled` | 已取消 |
